# obsidian-diff-apply 手工测试清单（按步骤逐项点检）

## 0. 冒烟（2 分钟）
### 0.1 启用/禁用插件
- 准备：Obsidian 打开任意 vault，启用本插件
- 操作：启用 → 等 5 秒 → 禁用 → 再启用
- 期望：无报错弹窗；界面无卡死；控制台无持续报错

### 0.2 入口是否出现
- 准备：打开一篇笔记
- 操作：选中一段文本 → 右键编辑器 → 找到菜单项（Hybrid Edit / Hybrid Diff）
- 期望：菜单项存在且可点击

---

## 1. 核心流程（必须全过）
### 1.1 无选区时的提示
- 准备：打开笔记，不选中文本
- 操作：运行命令面板里的 `Hybrid edit selection`（或同名命令）
- 期望：出现提示“先在笔记中选中原文”（类似文案）；不打开弹窗

### 1.2 正常打开（有选区 + 有剪贴板）
- 准备：复制一段文本到系统剪贴板；回到笔记选中一段原文
- 操作：运行命令或右键菜单打开弹窗
- 期望：
  - 左栏 Original 显示你选中的原文
  - 右栏 Modified 显示剪贴板内容
  - 中栏 Editor 初始为空/或按设计（以你当前实现为准），但可继续操作

### 1.3 Apply 替换范围正确
- 准备：在笔记中选中一段“中间位置”的文本（前后都有内容，便于确认范围）
- 操作：打开弹窗 → 让中栏产生一些最终文本（从左/右复制过来也行）→ 点击 Apply
- 期望：只替换原来那段选区；选区外内容完全不变

### 1.4 Cancel/Esc 不改动原文
- 准备：选中一段文本并打开弹窗
- 操作：在弹窗里做任意修改 → 按 `Esc` 或点 Cancel
- 期望：笔记内容完全不变

---

## 2. 剪贴板异常路径（必测一次）
### 2.1 剪贴板不可读/权限失败时仍可用
- 准备：在会触发剪贴板权限限制的环境里（或临时让读取失败）
- 操作：选中原文打开弹窗
- 期望：不崩溃；右栏可能为空；仍能从左栏复制到中栏并 Apply 成功

---

## 3. 复制/插入行为（中栏编辑体验）
### 3.1 左/右栏选区复制
- 准备：打开弹窗
- 操作：在左栏选中一段 → 触发“复制到 Editor”的动作（按钮/回车快捷键，按你的 UI）
- 期望：中栏插入的文字与选中内容完全一致；插入位置符合提示/设计（通常是光标处）

### 3.2 未选中时的提示
- 准备：打开弹窗，不在左/右栏选中任何文本
- 操作：触发复制动作
- 期望：提示“先选中要复制的文本”（类似文案）；中栏不发生变化

### 3.3 插入位置：行中不应强行加换行
- 准备：中栏已有一行文本，把光标放在行中间
- 操作：从左/右复制一段插入
- 期望：只插入文本本身；不会额外插入换行把文本顶到新行

### 3.4 插入位置：行首/行尾的智能换行
- 准备：中栏两行文本；把光标放在“某行行首”与“某行行尾”分别测试
- 操作：从左/右复制插入
- 期望：只在需要时补齐 1 或 2 个换行（符合你期望的段落分隔感）；不会多出 3+ 个空行

---

## 4. Diff 展示（可视回归）
### 4.1 Word 模式 vs Char 模式
- 准备：原文与改文准备一组“只改一个字母”和一组“只改一个词”的例子
- 操作：切换 Diff 粒度（Word/Char）
- 期望：
  - Word：以词为单位高亮变化更粗
  - Char：以字符为单位高亮更细
  - 切换后高亮立即刷新；不会残留上一次模式的高亮块

### 4.2 Hover 高亮
- 准备：原文/改文存在差异
- 操作：鼠标移动到差异区域附近（左右两栏都试）
- 期望：hover 高亮层按设计出现/消失；离开后恢复默认层

### 4.3 段落差异处加标注
- 准备：原文/改文存在段落行数差异
- 期望：原文及改文的段落间隙处出现编号标记

---

## 5. 滚动同步与长文本（稳定性）
### 5.1 左右滚动同步（长文本）
- 准备：原文/改文各至少 50 行
- 操作：滚动左栏到中部/底部；再滚动右栏
- 期望：overlay 高亮与文本行对齐；不会出现高亮“漂移”或明显错位

### 5.2 极端文本
- 准备：分别准备这些内容并测试打开弹窗：
  - 全空行/大量空行
  - 超长单行（> 2000 字符）
  - 含 CJK/emoji/标点混合
- 操作：打开弹窗、切换 diff 粒度、做一次复制插入、Apply
- 期望：不卡死；不崩溃；结果文本正确

---

## 6. 编辑模式与撤销（若你的 UI 有 Edit Mode）
### 6.1 Read Only / Edit Mode 切换
- 准备：打开弹窗
- 操作：切换到 Edit Mode，在中栏输入几字；再切回 Read Only
- 期望：Edit Mode 可输入；Read Only 不可输入（或按你的设计限制）

### 6.2 Undo/Redo
- 准备：中栏输入三次不同内容（分多次输入）
- 操作：`Cmd/Ctrl+Z` 连按；再 `Cmd/Ctrl+Shift+Z` 或 `Ctrl+Y`
- 期望：撤销/重做步进正确，不会跳步或丢历史

---

## 7. 样式/主题（发布前建议测）
### 7.1 深色/浅色主题
- 准备：切换 Obsidian 浅色/深色
- 操作：打开弹窗，查看 diff 高亮、按钮、文字对比度
- 期望：高亮可辨识；文字不糊；按钮 hover/focus 状态正常

### 7.2 窄窗口
- 准备：把 Obsidian 窗口缩窄
- 操作：打开弹窗，滚动、切换 diff 粒度、复制插入
- 期望：布局不崩；至少仍可操作并可 Apply

---

- Case B：只改 1 个词（测试 Word 粒度）

- Case D：删除一整行（测试 removed/added 的可视一致性）
- Case E：超长文本（性能冒烟）
